## 一、研究背景与动机

### 1.1 流式视频理解的重要性

随着直播、实时监控、自动驾驶等应用场景的快速发展，**流式视频理解（Streaming Video Understanding）** 的研究日益重要。与离线视频理解不同，流式视频理解只能在用户问题到达之前依赖已观察到的有限信息，因此如何高效处理持续输入的视觉信息成为关键挑战。

### 1.2 现有方法的局限

当前流式视频理解研究主要集中在两个方向：

| 方向 | 方法 | 局限性 |
|------|------|--------|
| **视觉Token压缩** | 在像素空间或特征编码后选择信息量大的token | 对极长视频仍会丢失关键视觉信息 |
| **KV Cache压缩** | 在LLM层面维护短期滑动窗口和长期记忆栈 | 难以维持实体（如人物身份）的长期一致性 |

**M3-Agent** 首次将长期Agent Memory引入流式视频场景，通过构建**记忆图（Memory Graph）** 来表示实体（人脸/声音节点）和视觉内容（文本节点），有效保持了信息完整性和实体一致性。

### 1.3 核心问题：记忆图的存储与检索瓶颈

随着记忆图规模增大，存储和检索效率成为主要瓶颈：

- 文本节点数量的增加导致**记忆检索延迟急剧上升**
- 论文中图1(b)展示：当文本节点数量从 10² 增长到 10⁶ 时，检索时间从几乎为零飙升至约20秒
- 这严重阻碍了对问题的实时回答

## 二、方法概述

**StreamMeCo**（Stream Agent Memory Compression）是一个**无需训练（training-free）** 的高效长期流式Agent记忆压缩框架，实现了首个工业级流式视频Agent的记忆压缩，并支持高效的记忆图检索。

框架由两大核心组件构成：
1. **记忆图压缩**：基于图结构的双分支压缩策略（EMsampling + EWpruning）
2. **记忆检索优化**：时间衰减记忆检索机制（TMR）

## 三、技术方法详解

### 3.1 预备知识：M3-Agent的记忆图结构

记忆图包含以下节点类型：

| 节点类型 | 描述 | 示例 |
|---------|------|------|
| **人脸节点（Face Nodes）** | 表示人物面部信息 | 人物身份标识 |
| **声音节点（Voice Nodes）** | 表示人物声音信息 | 声音特征 |
| **文本节点（Text Nodes）** | 抽象化的视觉内容描述 | "Mark午餐最喜欢的食物是米饭" |
| **边权重（Edge Weights）** | 节点间的关联强度 | 文本节点与人脸/声音节点的连接 |

**关键区分**：
- **孤立文本节点（Isolated Text Nodes）**：无实体引用，内容如"那个男人午餐最喜欢的食物是米饭"
- **连接文本节点（Connected Text Nodes）**：包含明确实体信息，如"Mark午餐最喜欢的食物是米饭"

记忆图检索分为两种方式：
- **基于角色的检索（Character-based）**：当查询包含"character id"时，返回k个最相似的文本节点
- **基于语义的检索（Semantic-based）**：按时间戳分段，每段选最相似的代表节点，最终返回两个最相似节点对应的所有文本内容

### 3.2 文本记忆压缩——双分支策略

#### 3.2.1 无边最小最大采样（Edge-free Minmax Sampling, EMsampling）

**适用对象**：孤立文本节点

**核心思想**：通过聚类+最大覆盖的方式选取最具代表性的文本节点

**具体步骤**：

**步骤1：球面KMeans聚类**

给定 N₁ 个孤立文本节点，使用 Spherical KMeans 算法根据文本节点内容的embedding向量将其聚类为 N₁×a 个簇：

$$\{C_j\} = \text{KMeans}(\{e_i\}, a)$$

其中 eᵢ 为文本节点内容的embedding向量，a 为聚类比例（控制簇的数量），Cⱼ 为第 j 个簇。

**步骤2：确定每簇保留数量**

假设孤立文本节点的保留比例为 α，则第 j 个簇保留 |Cⱼ| × α 个文本节点。

**步骤3：Minmax采样策略**

对第 j 个簇执行如下迭代选择过程：
1. 首先选择**距离簇中心最近的**文本节点，加入已选集合 Sⱼ
2. 计算每个未选节点到已选集合的**最小距离**：

$$D(u_i, S_j) = \min_{s_k \in S_j} \|e_{u_i} - e_{s_k}\|$$

3. 选择距离已选集合**最远的**未选节点加入 Sⱼ：

$$u_{i^*} = \arg\max_{u_i \in U_j} D(u_i, S_j)$$

4. 重复直到选中节点数量达到 |Cⱼ| × α

**设计直觉**：该策略确保被选中的节点在embedding空间中**均匀覆盖**整个簇，避免语义空洞（semantic holes），最大化信息保留。

#### 3.2.2 边感知加权剪枝（Edge-aware Weighting Pruning, EWpruning）

**适用对象**：连接文本节点（与人脸/声音节点有边连接的文本节点）

**核心思想**：综合考虑实体重要性和embedding相似度，选取最多样化且重要的文本节点

**具体步骤**：

**步骤1：计算实体重要性（Entity Importance）**

构建权重边矩阵 W ∈ ℝ^(M×N₂)，其中 M 为面/声节点总数，N₂ 为连接文本节点数。每个元素 wᵢⱼ 表示第 i 个实体节点与第 j 个文本节点之间的边权重。

对 W 进行行求和：

$$W' = \sum_{i=1}^{M} w_{ij}$$

W' ∈ ℝ^(1×N₂) 表示 N₂ 个连接文本节点相对于实体节点集合的重要性得分。归一化到 [0,1] 后得到最终的**实体重要性**。

**步骤2：计算Embedding相似度（Embedding Similarity）**

计算 N₂ 个文本节点的两两相似度矩阵 S ∈ ℝ^(N₂×N₂)，然后行求和：

$$S' = \sum_{i=1}^{N_2} s_{ij}$$

由于目标是选择**多样性更高**的节点，将 S' 归一化到 [0,1] 后取反：1 - S'，得到每个文本节点相对于连接文本节点集的embedding相似度（值越大表示越独特）。

**步骤3：计算融合得分（Fusion Score）**

对第 j 个连接文本节点，融合得分计算为：

$$r_j = b \cdot W'_j + (1 - b) \cdot S'_j$$

其中平衡系数 b 控制实体重要性和embedding相似度之间的权衡。

**步骤4：选择保留节点**

保留融合得分最高的 N₂ × β 个文本节点，剪枝其余节点。

### 3.3 时间衰减记忆检索（Time-decay Memory Retrieval, TMR）

**灵感来源**：人类记忆机制——近期记忆比远期记忆更清晰

**核心思想**：在检索阶段，根据时间衰减动态分配各时间段被检索的节点数量，优先从近期片段中检索更多记忆

**具体步骤**：

**步骤1：计算时间片段相关性**

根据查询内容embedding与所有文本节点的相似度，按时间戳将文本节点分为时间片段 Tⱼ (j=1,2,...,t)。每个时间片段的相关性得分为该片段内所有文本节点相似度的总和：

$$E_j = \sum_{i=1}^{|T_j|} s_{ij}$$

**步骤2：计算平均相似度并施加时间衰减**

$$\bar{E}_j = \frac{E_j}{|T_j|}$$

施加指数衰减函数模拟人类记忆衰退：

$$E'_j = \bar{E}_j \cdot e^{-\lambda(t - j)}$$

其中 λ 为记忆衰减系数，t 为当前时间步，j 为时间片段位置。

**步骤3：动态分配检索数量**

假设需要检索 k 个文本节点，从第 j 个片段选择的节点数量为：

$$Num_j = \frac{E'_j}{\sum_{i=1}^{t} E'_i} \times k$$

**步骤4：段内排序选择**

在每个时间片段内，按与查询的相似度排序，选择前 Numⱼ 个文本节点。

**关键优势**：TMR机制确保从更近期的视频片段中检索更多记忆，同时不完全忽略远期重要信息，在长期记忆和近期关键信息之间实现自适应平衡。

- **加入TMR后**：Avg-Retrieval显著降低，因为TMR优先选择更可靠、更高置信度的记忆条目
- 最终在所有数据集上实现平均 **1.87× 检索加速**

#### 5.2.4 超参数影响

**聚类比例 a 的影响**：
- a=0.05 时两个数据集均达到最佳性能
- a 过小 → 簇过大，语义差异大的节点被归入同一簇
- a 过大 → 簇过小，难以形成稳定的语义结构

**平衡系数 b 的影响**：
- b=0.1 时最佳
- b 过小 → 无法有效利用实体重要性信息
- b 过大 → 过度依赖实体重要性，忽略embedding相似度

- 指数衰减在较小衰减系数下即可达到强性能，更实用和鲁棒

## 六、理论分析

### 6.1 EMsampling的覆盖半径分析

在语义簇 C 内，EMsampling的贪心过程等价于逐步缩小覆盖半径：

$$\rho = \max_{x \in X} d(x, S) = \max_{x \in X} \min_{y \in S} \|x - y\|$$

当 ρ 较小时，每个原始节点 x 在 S 中都有一个近距离的代表节点，压缩后的节点**均匀覆盖**原始簇的embedding空间，避免语义空洞，理论上导致最小的准确率退化。

### 6.2 EWpruning的信息损失分析

根据评分函数 rⱼ = bW'ⱼ + (1-b)S'ⱼ，优先移除 rⱼ 较小的节点。压缩前后的总分损失：

$$\Delta X = \sum_{i=1}^{N} r_i - \sum_{j=1}^{M} r_j \approx 0$$

因此压缩引入的信息损失在理论上是最小的。

### 6.3 时间复杂度

| 模块 | 时间复杂度 |
|------|-----------|
| EMsampling（KMeans聚类） | O(N log N) |
| EMsampling（Minmax采样） | O(N²) |
| EWpruning（相似度矩阵+权重矩阵） | O(N²) |
| **总体** | **O(N²)** |

实际场景中，对30-60分钟的流式视频记忆图，每次压缩耗时**不到2秒**。

## 八、局限性

1. **数据集覆盖有限**：由于需要频繁调用 Gemini-2.5-Pro 和 text-embedding-3-large API，且记忆图生成耗时巨大，仅在三个数据集上进行了测试
2. **记忆图生成成本高**：依赖大量API调用和计算时间
3. **M3-Agent 当前仅对 M3-Bench-robot 和 M3-Bench-web 提供记忆图**

## 九、未来工作方向

1. **差异化自适应压缩**：为孤立节点和连接节点探索不同的压缩比例策略
2. **情景节点 vs 语义节点的差异化压缩**：情景节点关注特定事件，语义节点关注稳定知识，如何有效保留高判别力的事件描述和关键语义记忆是重要方向
3. **冗余与冲突记忆处理**：系统性研究和解决记忆图中的冗余、冲突甚至记忆中毒问题
4. **降低记忆图构建时间开销**：探索更高效的记忆构建机制

### 三大创新点

1. **Edge-free Minmax Sampling（EMsampling）**：基于球面聚类和最大覆盖策略，为孤立文本节点选取最具代表性的子集
2. **Edge-aware Weighting Pruning（EWpruning）**：融合实体重要性和embedding多样性，对连接文本节点进行智能剪枝
3. **Time-decay Memory Retrieval（TMR）**：受人类记忆机制启发，通过指数时间衰减动态分配检索配额，优先检索近期记忆

### 关键数据

- **压缩率 70%** → 检索加速 **1.87×**，平均准确率提升 **+1.0%**
- **压缩率 30% + TMR** → M3-Bench-robot 准确率从 30.3% 提升至 **34.6%**（+4.3%）
- 时间复杂度 **O(N²)**，实际运行 **<2秒**
- **Training-free**：无需额外训练，直接应用于已有记忆图
