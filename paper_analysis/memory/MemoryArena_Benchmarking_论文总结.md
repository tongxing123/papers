## 一、研究背景与动机

### 1.1 现有评估的两大缺陷

当前对Agent记忆的评估通常**孤立地**评估记忆化和行动：

| 评估类别 | 代表基准 | 评估内容 | 局限 |
|---------|---------|---------|------|
| **记忆化评估** | LoCoMo, LongMemEval | 回忆过去的对话或文本 | 无法捕捉记忆如何指导未来决策 |
| **行动评估** | WebArena, SWE-Bench | 单会话任务中的Agent行为 | 不需要长期记忆，不评估跨会话信息保留 |

### 1.2 现实场景的真实需求

> **在现实场景中，记忆化和行动是紧密耦合的**：Agent在与环境交互时获取记忆，随后依赖该记忆来解决未来任务。

这需要一个**记忆-Agent-环境循环（Memory-Agent-Environment Loop）**：
```
记忆更新 ← 环境反馈 ← Agent行动 → 环境
    ↑                                    ↓
    └──────────── 记忆检索 ←─────────────┘
```

### 1.3 核心观察

**在现有长上下文记忆基准（如LoCoMo）上接近饱和性能的Agent，在本文的Agentic设置中表现糟糕。**

> 当前基准的成功**并不能**有效转化为在目标驱动的交互式设置中使用记忆指导未来行动的能力。

## 三、四大评估环境

### 3.1 捆绑网购（Bundled Web Shopping）

**场景**：用户随时间购买相关产品，而非单次交易。后续购买依赖回忆之前购买的属性以确保兼容性和偏好一致性。

**示例**：购买相机套装
- 子任务1：购买相机机身（预算$800，选最便宜的）→ Canon EOS Rebel
- 子任务2：购买镜头（选评分最高的，**必须与Canon EOS兼容**）
- 子任务3：购买相机包（**必须与前面购买的设备兼容**）
- ...

**挑战**：Agent必须回忆之前购买了什么、识别兼容性约束、排除不兼容选项。

**构建方法**：基于WebShop环境扩展，利用类别层次结构识别兼容产品组，通过特征级兼容性推理构建接受-拒绝映射。

### 3.2 偏好约束规划：团体旅行（Group Travel Planning）

**场景**：初始旅行者规划行程，后续参与者逐步加入。团体成员可能共享活动（兴趣重叠），也可能在偏好分歧时要求个性化安排。

**约束类型**：
- **JOIN约束**："我想和Rebecca在第二天一起吃晚餐" → 必须分配相同活动
- **RELATION约束**："我想住评分比Rebecca的酒店至少高两级的酒店" → 相对偏好

**示例**：
- 子任务1：Tracy的5天德州行程（单人）
- 子任务2：Chelsea加入，喜欢豪华酒店，预算比Tracy多$150，第二天午餐要跟Tracy一起吃
- 子任务3：Emily加入...
- ...

**挑战**：每个子任务需要规划30个时间段行程，每个时间段受多重约束管辖。

### 3.3 渐进式网页搜索（Progressive Web Search）

**场景**：每个搜索步骤引入一个额外条件，最终答案必须满足所有之前引入的条件。

**示例**：
- 子任务1："1989年获得博士学位的人叫什么？" → 候选人列表A
- 子任务2："其中，2014年出版过书的人叫什么？" → 从A中筛选 → 列表B
- 子任务3："其中，2012年在会议上做过主旨演讲的人叫什么？" → 从B中筛选 → 最终答案
- ...

**挑战**：Agent必须保留并整合从早期搜索中获得的信息，每次搜索条件增加导致推理痕迹超过122k tokens。

### 3.4 顺序形式推理：数学与物理（Sequential Formal Reasoning）

**场景**：模拟研究级科学论文中的推理结构和难度。重大理论声明依赖涉及多个中间结果、定义和引理的长上下文论证。

**构建方法**：由高级博士级专家团队手工策划和标注学术论文，将中心声明分解为有序中间声明序列（引理和命题）。

**示例（数学）**：
- 子任务1：在Hermite分解中表达张量PCA的运行时上界
- 子任务2：使用前一推导计算积分Hermite多项式中的特定项
- 子任务3：推导更多结果...

**挑战**：验证单个声明可能需要数页推导，需要仔细复用之前建立的结论。

## 四、评估框架

### 4.1 记忆-Agent-环境循环

**单会话交互**：

$$a_{i,t} \sim \pi_A(\cdot | s, o_{i,1:t-1}, a_{i,1:t-1})$$

Agent在每个步骤根据当前指令和会话内交互历史选择动作。

**多会话交互**（需要持久记忆）：

$$m_{i,t} = \text{RETRIEVE}(M, s_i, a_{i,1:t-1}, o_{i,1:t-1})$$
$$a_{i,t} \sim \pi_A(\cdot | s_i, o_{i,1:t-1}, a_{i,1:t-1}, m_{i,t})$$
$$M \leftarrow \text{UPDATE}(M, (o_{i,1:T}, a_{i,1:T}))$$

### 4.2 评估的三类记忆系统

| 类别 | 记忆维度 | 代表系统 | 特征 |
|------|---------|---------|------|
| **长上下文Agent** | 0D | GPT-5.1-mini, GPT-4.1-mini, Gemini-3-Flash, Claude-Sonnet-4.5 | 原始历史追加，无抽象或整合 |
| **外部记忆Agent** | 1D/2D | Letta(MemGPT), Mem0, Mem0-g, ReasoningBank | 学习或启发式整合、蒸馏和检索 |
| **RAG系统** | 0D/1D/2D | BM25, Text-Embedding-3-Small, MemoRAG, GraphRAG | 索引文档存储 + 检索 |

### 4.3 评估指标

- **任务成功率（SR）**：完全解决的任务百分比
- **任务进度分数（PS）**：正确完成的子任务占比
- **软进度分数（sPS）**：每个子任务根据满足的约束比例获得部分分数（用于Group Travel Planning）
- **SR@k**：在第k个子任务处的成功率，衡量Agent在依赖加深时的性能衰减

## 五、实验结果

### 5.1 总体结果（GPT-5.1-mini为任务Agent）

| 方法类别 | 捆绑网购 SR | 团体旅行 SR | 渐进搜索 SR | 形式推理 SR(数学) | 形式推理 SR(物理) | 总平均SR |
|---------|-----------|-----------|-----------|----------------|----------------|---------|
| **长上下文Agent** | **0.06** | 0.00 | 0.04 | 0.38 | 0.45 | **0.16** |
| **外部记忆Agent** | 0.00 | 0.00 | 0.15 | 0.31 | 0.45 | **0.14** |
| **RAG系统** | 0.00 | 0.00 | 0.19 | 0.39 | 0.53 | **0.18** |
| **全部方法平均** | 0.02 | 0.00 | 0.09 | 0.35 | 0.42 | **0.15** |

**核心发现**：

1. **所有方法在MemoryArena中成功率都很低**：平均SR仅约15%
2. **团体旅行规划是最具挑战性的环境**：所有方法SR和PS均接近零
3. **PS远高于SR**：Agent能完成部分子任务，但无法将部分成功整合为全局一致的解决方案

### 5.2 关键发现详解

#### 发现1：外部记忆和RAG系统并非普遍有益

> **给GPT-5.1-mini配备外部记忆或RAG并不能一致地优于使用模型自身的长上下文历史。**

原因分析：
- **表示失配**：长上下文Agent在自洽的、逐字的交互历史上推理；外部记忆系统返回压缩、分段或重排序的信息
- **训练失配**：外部记忆系统未与任务Agent联合优化，Agent无法有效制定查询和整合检索信息

**"1+1 > 2"效应并不可靠。**

#### 发现2：外部记忆在特定场景下有帮助

外部记忆在以下场景产生一致性增益：
- **渐进式网页搜索**：单个子任务轨迹超过120k tokens
- **形式推理**：子任务需要高度复杂和领域特定的推理

在这些场景中，长上下文提示容易遭受**注意力饱和和错误累积**。外部记忆通过选择性抽象、蒸馏和保留任务相关信息来缓解这些失败模式。

#### 发现3：随依赖深度增加的性能衰减

所有评估方法都表现出**衰减趋势**，没有方法能在所有环境中维持一致的平坦区域：

- **渐进搜索**：长上下文Agent衰减更快（轨迹超122k tokens，超出有效上下文窗口）
- **形式推理/团体旅行**：基于检索的方法比外部记忆更鲁棒（需要精确复用早期信息时）

> 无论是长上下文模型还是现有外部记忆或检索机制，**都不足以可靠支持Agent在深度相互依赖子任务上的长视野执行。**

#### 发现4：延迟评估

| 方法类别 | 平均延迟(s) |
|---------|-----------|
| 长上下文Agent | **52.2s（最低）** |
| RAG系统 | 90.2-107.4s |
| 外部记忆Agent | **99.0-132.8s（最高）** |

**关键发现**：
- 长上下文Agent始终延迟最低，且在多个设置中性能有竞争力
- 记忆操作复杂度与执行延迟之间**没有系统性关系**
- 更复杂的记忆机制（2D）不一定产生更高延迟

### 5.3 POMDP视角

MemoryArena可自然视为**部分可观测马尔可夫决策过程（POMDP）**：

- Agent从未直接观测完整底层任务状态（潜在捆绑规格、约束演化、中间依赖）
- 每个会话仅接收部分观测（当前子任务指令 + 环境反馈）
- 外部记忆可解释为**信念状态估计的显式机制**

**性能衰减可解释为信念漂移**：Agent隐式状态估计中的小错误跨会话累积，最终主导下游决策。

**两个互补瓶颈**：

| 方面 | 瓶颈 |
|------|------|
| **记忆侧** | 当代记忆机制针对通用召回/压缩/语义相似性检索优化，保留和更新任务相关状态变量的能力有限 |
| **Agent侧** | 任务Agent未经训练以将记忆输出作为信念更新的结构化线索来查询、解释和整合 |

## 六、案例分析

### 6.1 捆绑网购：冲动购买导致下游预算失败

**GPT-5.1-mini（冲动选择）**：
- 搜索保湿霜 → 直接点击第一个选项（$19.99）→ 购买
- 错过了更优选项5（$13.45），多花$6.54

**Claude-4.5-sonnet（全面探索）**：
- 搜索 → 查看选项1（$19.99）→ 返回 → 探索选项2-4 → 查看选项5（$13.45）→ 购买最优

### 6.2 RAG因不精确检索而失败

**任务**：购买电视壁挂架（必须与之前购买的紧凑型Soundbar兼容）

**GPT-5.1-mini（长上下文）**：
- 能看到完整历史："我在Step 2购买了Compact Soundbar"
- 正确推理："Compact pairs well with Articulating" → 选择Articulating壁挂架 ✅

**GPT-5.1-mini + BM25 RAG**：
- 检索结果**缺少Step 2购买详情**
- 无法知道之前购买的Soundbar属性
- 错误选择Low Profile壁挂架 ❌

> **检索的精确性至关重要**：不完整的检索可能导致关键信息丢失。

## 八、核心贡献总结

### 8.1 三大贡献

1. **基准创新**：首个在多会话、相互依赖的Agent任务中统一评估Agent记忆的评估平台
   - 涵盖网页导航、偏好约束规划、信息搜索、形式推理

2. **评估发现**：
   - 在现有记忆基准上接近饱和的Agent在Agentic设置中表现糟糕
   - 外部记忆和RAG并非普遍有益
   - 所有方法在深度依赖下一致衰减

3. **理论视角**：将MemoryArena框架化为POMDP测试平台
   - 性能衰减 = 信念漂移
   - 外部记忆 = 信念状态估计机制

### 8.2 关键数据汇总

| 发现 | 数据支撑 |
|------|---------|
| 任务极具挑战性 | 所有方法平均SR仅~15%；团体旅行SR≈0 |
| 外部记忆不普遍有益 | Mem0/Mem0-g在多数环境不优于长上下文 |
| PS远高于SR | Agent能部分完成但无法全局一致 |
| 深度衰减 | 所有方法@k均显示衰减，无平坦区域 |
| 长上下文延迟最低 | 平均52.2s vs 外部记忆99-133s |
| 渐进搜索轨迹极长 | 平均122.4k tokens |
| LoCoMo饱和≠MemoryArena成功 | 在LoCoMo上表现优异的Agent在此表现差 |

## 九、对未来记忆系统设计的启示

### 9.1 两大互补瓶颈

| 瓶颈 | 描述 | 改进方向 |
|------|------|---------|
| **记忆侧** | 记忆机制针对通用召回优化，无法保留任务相关状态变量 | 设计感知任务依赖结构的记忆表示 |
| **Agent侧** | Agent未经训练以有效利用记忆输出 | 联合优化Agent训练目标与记忆表示 |

### 9.2 设计原则

1. **任务感知的记忆设计**：记忆系统应理解任务的因果依赖结构
2. **联合优化**：记忆机制与Agent应联合训练
3. **信念状态维护**：记忆应作为POMDP中的信念状态估计机制
4. **延迟-效果权衡**：更复杂的记忆不一定更好，需要考虑效率
5. **精确检索的重要性**：在需要精确复用信息的场景中，简单RAG可能优于复杂记忆

## 十、在同目录论文中的定位

| 论文 | 核心关注 | 本文与它的关系 |
|------|---------|-------------|
| NEMORI | 记忆蒸馏策略 | 被MemoryArena作为外部记忆系统评估 |
| MemP | 程序性记忆生命周期 | MemoryArena暴露了程序性记忆在多会话中的不足 |
| ReasoningBank | 推理策略+缩放 | 被MemoryArena评估，表现一般 |
| LEGOMem | 多Agent记忆分配 | MemoryArena可评估多Agent设置 |
| MemEvolve | 记忆架构元进化 | MemoryArena为元进化提供更具挑战性的评估 |

### MemoryArena的独特定位

> **这是一篇Benchmark论文**，与其他论文（提出方法）不同，它**揭示现有方法的不足**：
> 
> 1. **评估范式转换**：从"记忆回忆"到"记忆指导行动"
> 2. **任务设置创新**：强制跨会话因果依赖
> 3. **暴露核心差距**：LoCoMo饱和 ≠ 真实Agentic成功
